# Security Policy

## Supported Versions

Only the latest release on the `main` branch (production) is actively maintained and receives security fixes.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub/GitLab issues.**

If you discover a vulnerability in this project, report it privately by emailing:

**security@unc.edu**

Please include as much of the following as possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant URLs, request/response examples, or screenshots
- Your suggested fix, if you have one

## What to Expect

- **Acknowledgment** within 2 business days
- **Status update** within 7 business days with an assessment and planned timeline
- Credit in the release notes if you'd like it, once the issue is resolved

We appreciate responsible disclosure and will work with you to address the issue before any public announcement.

## Scope

This policy applies to the `whatismyip` application code in this repository. Vulnerabilities in third-party dependencies should be reported to their respective maintainers.
